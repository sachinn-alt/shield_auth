How it works:

  • Header: Specifies the cryptographic algorithm and token type.
  • Payload: Contains public claims about the user (ID, Email, Username, Expiry).
  • Signature: The backend verifies the token integrity using a private signing key. If even a single character in the token is altered by the client, the backend instantly rejects it as untrusted.
